# QueryDeck Support

QueryDeck is early and moving quickly. The fastest way to get help or report a problem is to open a GitHub issue with enough detail to reproduce the workflow.

## Before Opening An Issue

- Check that you are using the latest Marketplace or Open VSX version.
- Reload VS Code after updating the extension.
- Run the failing workflow again from a fresh query console when possible.
- Remove private hostnames, database names, credentials, customer data, and production SQL from screenshots or logs.

## Useful Issue Details

Include:

- QueryDeck version.
- VS Code, Cursor, or compatible editor version.
- Operating system.
- Database engine and version, such as PostgreSQL 16 or MySQL 8.
- Whether the connection is local, SSH-tunneled, cloud-hosted, or warehouse-backed.
- The command or UI action that failed.
- Sanitized SQL, error text, and screenshots when useful.

## Security Or Privacy Reports

Do not post secrets or sensitive connection details in public issues. For security-sensitive reports, use the guidance in [SECURITY.md](SECURITY.md).
