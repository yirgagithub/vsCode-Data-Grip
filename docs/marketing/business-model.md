# QueryDeck Business Model

## Near-Term Goal

Grow installs, developer trust, and feedback. QueryDeck should stay free while the workflow is still becoming credible.

## Money Path

### Free Extension

The free extension builds distribution:

- Multi-database SQL workbench.
- Schema explorer.
- Query consoles and result grids.
- Local query memory.
- AI explain/fix SQL.
- Read-only MCP server.

### Paid Team Product

Potential paid features once individual usage is proven:

- Shared team query library.
- Team query memory with access controls.
- Centralized connection profiles without sharing passwords.
- Audit logs for AI-agent database context.
- Workspace-level MCP gateway with policy controls.
- Schema documentation and query runbooks generated from local/team history.
- Admin controls for production safety policies.

### Services Revenue

Services can generate cash earlier than SaaS:

- AI-ready database workflow setup for engineering teams.
- Safe MCP configuration for internal databases.
- Migration from scattered SQL files and desktop tools into QueryDeck workflows.
- Query performance review and runbook setup.

## Buyer Profiles

- Engineering teams adopting Cursor, Claude Code, Codex, or other AI coding agents.
- Data-heavy startups that do not want to buy seats for heavier database IDEs.
- Backend teams that need safer production SQL workflows inside VS Code.
- Analytics engineers who repeatedly reuse and modify query patterns.

## What Not To Sell Yet

- Hosted sync before security and team controls are designed.
- Enterprise claims without support process and security posture.
- "Replaces every database IDE" claims.
- AI autopilot claims for write queries.

## First Revenue Experiment

Offer a fixed-price setup package:

```text
AI-ready database workflow setup for VS Code/Cursor teams.

Includes QueryDeck setup, read-only database roles, MCP configuration, sample query memory, safety settings, and a team runbook.
```

Package ideas:

- Starter setup: one database, one team, one workflow.
- Production safety setup: read-only roles, destructive query prompts, MCP row limits.
- Analytics workflow setup: query library, parameterized templates, history search conventions.
