=====================================================================
SKETCH 1: TABLE PERFORMANCE ADVISOR
=====================================================================

Step 1 — new driver method (src/database/drivers/DatabaseDriver.ts):

    getTableStats(connectionId: string, schema: string, table: string): Promise<TableStatsInfo>;

  Postgres impl:
    - pg_stat_user_tables: seq_scan vs idx_scan, n_live_tup, n_dead_tup,
      last_vacuum/last_analyze
    - pg_stats: per-column n_distinct, null_frac, correlation
      (correlation -> BRIN index / clustering candidates)
  Redshift override (one query gives everything):

    SELECT "table", diststyle, sortkey1, sortkey_num, size, tbl_rows,
           skew_rows, unsorted, stats_off, encoded
    FROM svv_table_info
    WHERE schema = $1 AND "table" = $2;

Step 2 — workload aggregation:
  New method QueryMemoryService.getTableWorkload(connectionId, 'schema.table'):
  filter QueryMemoryItems whose tables include target, rank by
  runCount * durationMs, parse top ~15 SQLs for columns used in
  JOIN / WHERE / GROUP BY / ORDER BY. Plain code, no AI.

Step 3 — deterministic pre-pass (keeps the LLM honest):
  - skew_rows > 4        -> distribution skew
  - unsorted > 20        -> needs VACUUM SORT
  - stats_off > 10       -> stale stats, needs ANALYZE
  - seq_scan >> idx_scan on large table -> missing index
  AI explains/prioritizes these findings, doesn't invent them.

Step 4 — new adapter method adviseTablePerformance(request), following the
  summarizeQueryMemory JSON pattern. Prompt gets: table DDL (getTableDDL),
  stats, pre-pass flags, workload summary ("joined on user_id in 12 queries
  totaling 340s; filtered on created_at in 9"). Returns:

    { "findings": ["..."],
      "recommendations": [
        { "kind": "sortkey|distkey|index|partition|vacuum|analyze",
          "rationale": "...", "ddl": "ALTER TABLE ...",
          "impact": "high|medium|low" } ] }

Step 5 — UI:
  - Register database.analyzeTablePerformance on table context menu
    (next to "Open Table Data")
  - v1: render in TableDataPanel (template string, cheap to extend) with
    "copy DDL" / "open in console" per recommendation
  - NEVER auto-execute; generated DDL goes through SqlSafetyClassifier
    when the user runs it.

Size: ~5 files (driver interface + 2 drivers, adapter, service, command/panel)
+ tests.

SKETCH 3: VISUAL EXPLAIN (after the above)
=====================================================================

- Driver: explainQuery(params, { analyze?: boolean }) -> EXPLAIN (FORMAT JSON);
  ANALYZE behind a confirmation (it executes the query).
- New tab kind in results panel: plan tree instead of grid.
- AI annotates top-cost nodes (seq scans on big tables, bad nested loops,
  sort spills) with plain-English fixes + rewritten SQL.
- Biggest UI lift (tree component) — sequence last.
